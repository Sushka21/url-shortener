# URL-Shortener

## Архитектура проекта
* `cmd/` — точка входа в приложение
* `internal/entity/` — бизнес объекты и ошибки
* `internal/usecase/` — бизнес логика
* `internal/repository/` — слой для работы с хранилищами данных (Postgres / In-Memory)
* `internal/controller/` — слой HTTP-хендлеров для приема и обработки запросов


Для удобства локальной разработки все рутинные команды вынесены в `Taskfile`.

## Локальный запуск
* `task up` — запуск сервиса
* `task down` - завершение работы сервиса

## Тестирование
* `task test` - запуск интеграционных тестов
* `task unit-test` - запуск юнит тестов(перед запуском нужно выполнить комаду 
task generate для генерации моков)


## Дополнительные скрипты
* `task lint` — запуск линтера
* `task generate` — запуск генарции кода
* `task build` — компиляция приложения (в папку `./bin/app`)
